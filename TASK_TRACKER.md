## Known bugs

- **Лайки чужих постов через API**: серверный баг `Notification::emit()` — лайк может не сохраняться (HTML 500). Клиентский фикс: `execute()` требует валидный JSON, лайк откатывается.
- **Обсуждения: авторы-группы**: `board.getComments` отдаёт club id в `profiles` без аватарки → ручная подгрузка `groups.getById`.
- **Обсуждения: virtual_id**: `board.getTopics` → DB id, `getComments` → virtual_id. Guess-and-verify workaround.
- **Длинные очереди сообщений**: OpenVK отдаёт ТОЛЬКО ОДНО событие на запрос → «средние» события проглатываются.
- **PM attachments**: API не отдаёт вложения в `getHistory`.
- **chat_id → fail 946**: чаты только 1-на-1.

## OpenVK link-аудит (2026-07-09)

> Полная актуальная карта маршрутов вынесена в `LINK_ROUTES.md`. Таблица ниже — исторический аудит.


Форматы ссылок проверены по серверному коду OpenVK (`Web/Models/Entities/*::getURL`, `Post::getPrettyId`, `Video::getPrettyId`, `Playlist::getURL`):
- **OpenVK audio-альбомы = `/playlist`**, НЕ `/album` (`Playlist::getURL` → `/playlist{ownerRealId}_{id}`). iOS-парсер уже ловит `playlist` — ок.
- **Кастом-инстанс НЕ перехватывается**: `LinkParser.hosts` жёстко привязан к `openvk.org/.xyz` + `ovk.to`. На любом другом `instance.webURL` ссылки НЕ перехватываются вообще → уходят в браузер. Нужно сверять с `settings.instance`, а не хардкодить хосты.
