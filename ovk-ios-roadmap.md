- **Поиск треков — важный нюанс:** `audio.search`→`Audios::search` использует MySQL `MATCH ... AGAINST(... IN BOOLEAN MODE)` (поиск по ЦЕЛЫМ словам), поэтому «jew» не находит «jew3ss». Решение: в SearchViewModel к каждому слову добавляем wildcard `*` (`jew`→`jew*`). Альбомы (`audio.searchAlbums`→`findPlaylists`) используют `LIKE %query%` (подстрока) — там wildcard НЕ нужен и сломал бы (попал бы литералом в LIKE).

- **БАГ ownerID=0 (2026-07-01):** симптомы — стена пустая + video.get даёт HTTP 400. Причина: `isLoggedIn = token != nil` (Keychain переживает переустановку), а `settings.userID` (UserDefaults) мог потеряться → nil → `ownerID = settings.userID ?? 0` = 0 → wall.get owner_id=0 (пусто), video.get owner_id=0 → `fail(1,"Not implemented")` = HTTP 400. Профиль-ШАПКА при этом грузилась (users.get user_ids=0 = текущий, не зависит от userID). Фикс: `ownerID = model.user?.id ?? ...` (id из загруженного профиля надёжнее) + `AppSettings.rememberUserID()` дозаписывает userID из загруженного своего профиля. УРОК: не полагаться на settings.userID для owner_id — брать из users.get.
- **Видео готово (2026-07-01):** OpenVK отдаёт нативные видео как **прямой mp4** в `files.mp4_480` (=getURL()), внешние (youtube) — `files=null`, `player`=ссылка, `platform="youtube"`. `Video` декодит `files`([String:String], берём макс. качество mp4_1080..240), `player`, `platform`, width/height. `VideoPlayerScreen` (fullScreenCover из VideosView по тапу): нативное → `AVKit.VideoPlayer(AVPlayer(url:mp4))` автоплей; внешнее → `WebView`(WKWebView) с `player` URL. Музыка ставится на паузу при открытии видео. Если нативное mp4 не заиграет — вероятно, файл на greyweb-домене → нужны cookie/UA через AVURLAsset (пока не делал).
- **ЗВУК видео + VLCKit (2026-07-01):** OpenVK кодирует нативное видео как **H.264 + MP3-в-MP4** (см. `Web/Models/shell/processVideo.sh`: `-c:v libx264 -c:a libmp3lame`). Apple AVFoundation/Safari/WKWebView **НЕ декодят MP3 внутри MP4** → видео есть, звука нет (и в браузере тоже). Единственный фикс — не-Apple декодер. Подключён **MobileVLCKit 3.6.0** (динамический xcframework, ~1.1ГБ на диске / ~120МБ в бандле): лежит в `Vendor/MobileVLCKit.xcframework` (в .gitignore, качается с download.videolan.org/cocoapods/prod). Прописан в project.yml: `dependencies: framework embed:true codeSign:true` + системные sdk-фреймворки (AVFoundation,AudioToolbox,CoreMedia,CoreVideo,VideoToolbox,CoreGraphics,CoreText,QuartzCore,OpenGLES,Security,CFNetwork,MediaPlayer,SystemConfiguration + libc++/iconv/xml2/z/bz2.tbd) + `CLANG_CXX_LIBRARY=libc++`. `VLCVideoView` (UIViewRepresentable над VLCMediaPlayer) играет нативные видео (streamURL) со звуком; youtube → WebView. Собирается на Xcode 14.3.1, фреймворк встраивается. Звук подтверждён рабочим. Плеер с контролами: `VLCPlayerController` (ObservableObject+VLCMediaPlayerDelegate, публикует isPlaying/isBuffering/currentSeconds, делегат зовёт с потока VLC → DispatchQueue.main), `VLCVideoSurface` (UIViewRepresentable drawable). `VideoPlayerScreen`: тап — показать/скрыть контролы, центр — play/pause, снизу — Slider с таймкодами (перемотка через player.position), спиннер при буферизации. Длительность берём из video.duration (VLCTime.length бывает 0 в начале). Делегат-методы должны быть `Notification` (НЕ `Notification!`) — иначе warning optionality. Буферизация: `isBuffering` true только пока `!player.isPlaying` + сбрасывается в mediaPlayerTimeChanged (иначе колёсико висит поверх играющего видео). **Поворот экрана:** приложение по умолчанию портрет (VK-стиль), для видео — ландшафт. Реализовано через `AppDelegate` (`@UIApplicationDelegateAdaptor`) с `static orientationLock` + `application(_:supportedInterfaceOrientationsFor:)`; project.yml UISupportedInterfaceOrientations теперь включает landscape; `VideoPlayerScreen.onAppear/onDisappear` зовёт `AppDelegate.setVideoOrientation(true/false)`. ПАССИВНЫЙ поворот (телефоном) на iOS15 из SwiftUI ненадёжен + не работает при блокировке автоповорота в Пункте управления, поэтому добавлена КНОПКА поворота в контролах видео → `AppDelegate.forceRotate(to:)`: iOS15 `UIDevice.setValue(orientation,forKey:"orientation")`+`UIViewController.attemptRotationToDeviceOrientation()`, iOS16 `requestGeometryUpdate`. Работает независимо от rotation lock. Если VLC не скачает файл — добавить cookie/UA через VLCMedia options.

В сессии работало, т.к. чип берётся из ОПТИМИСТИЧНОГО react() (по реальному message.id, без decode).
